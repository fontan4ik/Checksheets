# Документация по API ЭТМ (ETM API)

## 1. Общие сведения

### 1.1. Доступ и Авторизация
Для работы с API необходимо иметь регистрационные данные в системе iPro (логин и пароль).
*   **Production сервер:** `https://ipro.etm.ru/api/v1`
*   **Test сервер:** `https://itest2.etm.ru/api/v1`

**Авторизация:**
Перед отправкой любых запросов необходимо получить ключ сессии.
*   **Метод:** `POST`
*   **URL:** `/user/login`
*   **Параметры:**
    *   `log` (M): Логин пользователя.
    *   `pwd` (M): Пароль пользователя.
*   **Ответ:** JSON. В поле `data.session` содержится ключ авторизации.
*   **Время жизни ключа:** 2 часа.
*   **Пример запроса:**
    ```http
    POST https://ipro.etm.ru/api/v1/user/login?log=ВАШ_ЛОГИН&pwd=ВАШ_ПАРОЛЬ
    ```
*   **Пример ответа:**
    ```json
    {
      "status": { "code": 200, "message": "" },
      "data": {
        "session": "110377745112238593650805959783",
        "basket_lines": 0,
        "url": "s2.etm.ru"
      }
    }
    ```

### 1.2. Ограничения и частота запросов
*   **Авторизация:** 1 запрос в 2 минуты.
*   **Характеристики/Цены/Остатки:** 1 запрос в секунду.
*   **Блокировка:** При превышении частоты запросов IP клиента может быть заблокирован.
*   **Иностранный хостинг:** Требуется официальное письмо на имя гендиректора ЭТМ с указанием внешних IP.

### 1.3. Общие параметры запросов
*   **session-id:** Ключ сессии, полученный при авторизации. Обязателен для всех методов кроме авторизации.
*   **Типы кодов товаров:**
    *   `cli` — коды клиента (по умолчанию). Требуется предварительное сопоставление с кодами ЭТМ.
    *   `etm` — коды ЭТМ (только цифры, без префикса).
    *   `mnf` — артикулы производителя.
*   **Статусы ответов (Status Code):**
    *   `200` — Ок (Успех).
    *   `400` — Ошибка (неверные параметры, данные не найдены).
    *   `403` — Ошибка проверки сессии (истекла или неверна).
    *   `404` — Объект не найден.

### 1.4. Форматы полей
*   `M` — Обязательное поле.
*   `O` — Опциональное поле.
*   `C` — Обязательное при определенных условиях.
*   `N` — Число.
*   `S` — Строка.
*   `Date` — Дата (CCYY-MM-DD или DD/MM/YYYY в зависимости от метода).

## 2. API Товаров (Product API)

### 2.1. Price (Цены)
Получение текущих цен по выбранной позиции.
*   **Метод:** `GET`
*   **URL:** `/goods/{КодТовара}/price`
*   **Параметры:**
    *   `type` (O): Тип кода (`cli`, `etm`, `mnf`). По умолчанию `cli`.
    *   `session-id` (M): Ключ сессии.
*   **Список товаров:** Коды можно передавать через `%2C` (до 50 кодов в запросе).
*   **Пример запроса:**
    ```http
    GET https://ipro.etm.ru/api/v1/goods/9536092/price?type=etm&session-id=КЛЮЧ
    ```
*   **Пример ответа:**
    ```json
    {
      "status": { "code": 200, "message": "" },
      "data": {
        "gdscode": 9536092,
        "price": 679.51,          // Цена без НДС
        "pricewnds": 815.41,      // Цена с НДС
        "price_tarif": 1288.16,   // Цена по тарифу производителя
        "price_retail": 1288      // Розничная цена
      }
    }
    ```
    *Примечание:* Если все цены равны 0, цена предоставляется по индивидуальному запросу.

### 2.2. Remains (Остатки)
Существует два метода получения остатков.

#### Метод 1: Остатки по конкретному товару
*   **Метод:** `GET`
*   **URL:** `/goods/{КодТовара}/remains`
*   **Параметры:** `type` (cli/etm/mnf), `session-id`.
*   **Пример ответа:**
    ```json
    {
      "status": { "code": 200, "message": "" },
      "data": {
        "gdscode": "8623260",
        "UnitName": "шт",
        "InfoStores": [
          {
            "StoreCode": 35,
            "StoreType": "rc", // rc - Региональный центр, crs - ЛЦ, op - Офис продаж
            "StoreName": "Центр. склад, г.Воронеж",
            "StoreQuantRem": 32
          }
        ],
        "InfoSuppStores": [], // Остатки на складе производителя
        "InforDeliveryTime": {
          "DeliveryTimeInPres": "8", // дней при наличии
          "DeliveryProductionTerm": "7 дн." // срок изготовления при отсутствии
        }
      }
    }
    ```

#### Метод 2: Выгрузка остатков всех товаров на складе ЛЦ
*   **Метод:** `GET`
*   **URL:** `/goods/remains`
*   **Параметры:**
    *   `store` (O): Код РЦ ЭТМ (например, 11, 12, 14). Если не указан, выгрузка по всем складам клиента (макс 2 склада одновременно).
    *   `session-id` (M).
*   **Рекомендация:** 1 склад = 1 запрос.
*   **Пример запроса:**
    ```http
    GET https://ipro.etm.ru/api/v1/goods/remains?store=11&session-id=КЛЮЧ
    ```
*   **Пример ответа:**
    ```json
    {
      "status": { "code": 200, "message": "" },
      "data": {
        "rows": [
          {
            "StoreCode": "11",
            "GdsCode": "7421",
            "Article": "PSN21-016-3",
            "RemInfo": "30"
          }
        ]
      }
    }
    ```
*   **Коды распределительных комплексов (РЦ):**
    *   11: РЦ СПБ, 12: РЦ Урал, 13: РЦ Самара, 14: РЦ Москва, 15: РЦ ЮГ, 16: РЦ Сибирь, 17: РЦ Казань, 18: РЦ Чехов, 19: РЦ Малоярославец, 35: РЦ Воронеж, 36: ЛЦ Владивосток.

### 2.3. Goods (Характеристики)
Получение подробной информации о товаре (характеристики, изображения, аналоги).
*   **Метод:** `GET`
*   **URL:** `/goods/{КодТовара}`
*   **Параметры:**
    *   `id` (M): Код товара.
    *   `type` (M): Тип кода (cli/etm/mnf). По умолчанию `etm`, если не заполнено.
    *   `session-id` (M).
*   **Ключевые поля ответа:**
    *   `gdsNameTitle`: Полное наименование.
    *   `gdsMnfName`: Производитель.
    *   `gdsArt`: Артикул.
    *   `gdsChars`: Массив характеристик (Наименование, Значение).
    *   `gdsPacks`: Упаковки (вес, габариты: длина, ширина, высота, объем).
    *   `gdsImages`: Ссылки на изображения (`gdsImgSrc` - малое, `gdsImgRef` - стандартное). Сервер: `https://cdn.etm.ru`.
    *   `barcodes`: Штрихкоды.
    *   `relatedProducts`: Аналоги/взаимозаменяемые товары.
    *   `certificates`: Сертификаты (ссылки на файлы).
*   **Пример запроса:**
    ```http
    GET https://ipro.etm.ru/api/v1/goods/9536092?type=etm&session-id=КЛЮЧ
    ```

### 2.4. SgGds (Список номенклатуры)
Получение списка товаров по согласованному спецпрайсу. Процесс асинхронный.
*   **Шаг 1: Создание задачи**
    *   **Метод:** `POST`
    *   **URL:** `/job/create/40029846`
    *   **Параметры:** `session-id`.
    *   **Ответ:** Содержит `uuid` задачи.
*   **Шаг 2: Получение результата**
    *   **Метод:** `GET`
    *   **URL:** `/job/{UUID}`
    *   **Параметры:** `session-id`.
    *   **Статусы задачи (`state`):** 0 - создана, 1 - успешно завершена, 2 - ошибка, 3 - ждет запуска.
    *   **Обработка:** До 3 часов. Если задача не готова, повторять запрос через 1 час (с новым session-id и тем же UUID).
    *   **Ответ (при успехе):** Содержит `urls` — ссылку на JSON файл со списком товаров.
*   **Структура файла номенклатуры:**
    *   `id`: Код ЭТМ.
    *   `name`: Наименование.
    *   `brand`: Бренд.
    *   `article`: Артикул.
    *   `cli_code`: Код клиента.

### 2.5. Info (Справочники)
Получение справочника производителей.
*   **Метод:** `GET`
*   **URL:** `/info/search/r-manuf/`
*   **Параметры:** `session-id`.
*   **Ответ:** Массив `rows`, где `id` — внутренний номер ЭТМ, `value` — код производителя, `label` — наименование.

## 3. API Заказов (Order API)

### 3.1. Размещение заказа (Create)
Создание нового заказа в системе ЭТМ.
*   **Метод:** `POST`
*   **URL:** `/invoice/create`
*   **Формат тела запроса:** JSON.
*   **Основные поля запроса:**
    *   `OrderNumber` (M): Номер заказа в системе клиента (до 16 знаков).
    *   `OrderDate` (O): Дата заказа (CCYY-MM-DD).
    *   `DocumentFunctionCode` (M): Тип документа (`P` - первичный, `A` - подтверждение, `D` - отказ, `C` - изменение).
    *   `Seller/ILN` (M): GLN продавца (константа `4660011519999`).
    *   `Order-Lines` (M): Массив товарных позиций.
        *   `LineNumber`: Номер строки.
        *   `SupplierItemCode`: Код товара ЭТМ (или `BuyerItemCode`/`ManufacturerArticle`).
        *   `OrderedQuantity`: Количество.
        *   `UnitOfMeasure`: Ед. измерения (ОКЕИ, например, PCE).
        *   `OrderedUnitNetPrice`: Цена без НДС.
*   **Ответ:**
    *   `data.id`: ID заказа в системе ЭТМ (например, `1-2065278809`).
*   **Важно:** Если заказ меняется (`DocumentFunctionCode` = C), система аннулирует старый и создает новый с новым ID. Изменение возможно только для статусов "Спецификация" или "Счет на оплату".

### 3.2. Проверка статуса заказа (General Status)
Получение общей информации по заказу (без товарных строк).
*   **Метод:** `GET`
*   **URL:** `/invoice`
*   **Параметры:**
    *   `usr-inv-num` (O): Номер заказа клиента.
    *   `D1`, `D2` (O): Период (Дата от/до).
    *   `status` (O): Код статуса документа.
    *   `sessionID` (M).
*   **Ответ:** Массив `rows`. Один заказ клиента может быть расщеплен на несколько заказов ЭТМ (параметр `records` покажет количество).
*   **Поля ответа:**
    *   `id`: ID заказа ЭТМ.
    *   `status_code`: Код статуса (см. справочник).
    *   `status_name`: Наименование статуса.
    *   `summa`: Сумма документа.
    *   `invWhomcre`: ФИО инициатора заказа.

### 3.3. Состав заказа (Order Body)
Получение детальной информации по товарам в заказе ЭТМ.
*   **Метод:** `GET`
*   **URL:** `/invoice/{id}/body`
*   **Параметры:**
    *   `id` (M): ID заказа в системе ЭТМ (получается из метода `/invoice`).
    *   `sessionID` (M).
*   **Ответ:** Массив `rows` с товарными позициями.
*   **Поля ответа:**
    *   `gdscode`: Код товара.
    *   `cnt`: Количество.
    *   `price`: Цена с НДС.
    *   `dateotgr`: Планируемая дата отгрузки.
    *   `st_code`: Код склада отгрузки.
    *   `gdsImage`: Ссылка на изображение.
    *   `weight`, `volume`, `length`, `width`, `height`: Габариты и вес.

### 3.4. Печатная форма счета
*   **Метод:** `POST`
*   **URL:** `/invoice/{id}/print/bill`
*   **Параметры:** `session-id` в URL.
*   **Ответ:** Файл формата PDF (ссылка на файл).
*   **Ограничение:** Недопустимо для статуса `01` (Спецификация). Допустимо для `05` (Счет на оплату) и других.

### 3.5. Доставка и получение заказа

#### Создание точки доставки
*   **Метод:** `POST`
*   **URL:** `/delivery/point/create`
*   **Параметры:**
    *   `t-adr` (M): Адрес.
    *   `t-gps` (M): Координаты (широта, долгота, 6 знаков после запятой).
    *   `t-tel` (M): Телефон.
    *   `t-kont` (O): ФИО контактного лица.
    *   `t-info` (O): Информация для водителя.
*   **Ответ:** ID точки доставки.

#### Доступные даты отгрузки
*   **Метод:** `GET`
*   **URL:** `/delivery/point/{id}/date`
*   **Параметры:**
    *   `id` (M): ID точки доставки.
    *   `month`, `year` (O): Месяц и год.
    *   `sessionID` (M).
*   **Ответ:** Список доступных дат (`shipmentDates`).

#### Заявка на самовывоз
*   **Метод:** `POST`
*   **URL:** `/invoice/{id}/order`
*   **Параметры:**
    *   `id` (M): ID заказа ЭТМ.
    *   `skl` (M): Код офиса продаж ЭТМ.
    *   `sessionID` (M).

#### Заявка на доставку
*   **Метод:** `POST`
*   **URL:** `/invoice/{id}/delivery`
*   **Условие:** Доступно только для статуса заказа `05` (Счет на оплату).
*   **Параметры:**
    *   `id` (M): ID заказа ЭТМ.
    *   `adr` (M): ID точки доставки.
    *   `data` (M): Желаемая дата (DD/MM/YYYY).
    *   `time`, `time_po` (M): Время разгрузки (с/по).
    *   `phone` (M): Телефон.
    *   `session-id` (M).

### 3.6. Справочник статусов документа
| Код | Наименование |
| :--- | :--- |
| 01 | Спецификация |
| 05 | Счет на оплату |
| 10 | Расторгнутый счет |
| 15 | Резервы |
| 20 | Заказ у производителя |
| 24 | В подборе на складе ЛЦ |
| 27 | Отправлено на склад |
| 40 | Готов к выдаче |
| 45 | Отгружено |
| 60 | Возврат от клиента |

## 4. Рекомендации для разработки (Context for AI)

1.  **Управление сессией:** Всегда проверяйте валидность `session-id`. Если получаете ошибку `403`, необходимо повторно выполнить авторизацию (`/user/login`).
2.  **Обработка заказов:** Помните, что один заказ клиента (`OrderNumber`) может быть разбит системой ЭТМ на несколько внутренних заказов (`id`). При проверке статуса всегда анализируйте массив `rows` в ответе `/invoice`.
3.  **Асинхронность:** Метод выгрузки номенклатуры (`SgGds`) требует реализации механизма опроса статуса задачи по `UUID`.
4.  **Кодировка:** Все текстовые данные должны быть в UTF-8. Специфические символы в URL (например, запятая в списке кодов товаров) должны быть urlencode (`%2C`).
5.  **Габариты:** В методах `Goods` и `Order Body` габариты могут возвращаться в разных единицах измерения (метры, кг, литры), проверяйте поля `Dim` (единица измерения) там, где они доступны.
