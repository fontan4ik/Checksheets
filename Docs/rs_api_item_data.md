1.	Общие сведения
API возвращает ответ в формате JSON.
При работе с API используется Basic авторизация, в качестве данных авторизации используются логин и пароль от RS24.ru.
Параметры авторизации:
Необходимо передавать заголовок Authorization со значением:
Basic + переведенная в Base64 строка, содержащая логин и пароль, разделенные символом
«двоеточие».
Например, для пары логина LOGINTEST и пароля Pass123456 строка «LOGINTEST:Pass123456» в Base64 будет выглядеть так: TE9HSU5URVNUOlBhc3MxMjM0NTY=
А заголовок целиком будет выглядеть следующим образом:
Authorization:Basic TE9HSU5URVNUOlBhc3MxMjM0NTY=
Пример использования онлайн-декодировщика для получения логина и пароля в Base64:


Для каждого сервиса действует ограничение на получение данных - 150 запросов/30 сек с одного IP адреса. При превышении данного лимита пользователю будет заблокирован доступ к соответствующему сервису на 1 час. В случае блокировки будет возникать ошибка с кодом 403.

Общая схема работы:
1.	Получаем список складов с идентификаторами, используя API со складами. Эта информация потребуется для получения списка позиций и информации об остатках.
2.	Используя API со списком позиций, получаем список интересующей нас номенклатуры(складской/заказной/всей) по нужному нам складу. Эта информация нужна для получения данных об остатках, ценах и технических характеристиках позиции
3.	В зависимости от того, какая информация требуется, используем API с ценами, остатками, характеристиками, для получения соответствующих данных по интересующему нас артикулу.

3.	API Список позиций
Данный метод используется для получения списка позиций.
Для получения списка необходимо с методом GET обратиться по URL: https://cdis.russvet.ru/rs/position/*id склада*/*категория позиций* Параметры:





«Категория позиций» определяет, какие позиции будут выгружаться в ответе. Допустимые значения:
•	instock – для получения списка позиций, имеющихся в наличии на выбранном складе
•	custom – для получения списка позиций, которые привозятся под заказ
•	custcode – чтобы получить список связанных позиций в случае, если в системе Русского Света заведены связки позиций с клиентскими номенклатурными кодами.

4.2	Массовая выгрузка цен

Если в теле запроса будет передано более 50 кодов, то в ответе будет выведена информация о 50 кодах, остальные будут проигнорированы. На вход необходимо подавать коды позиций Русского Света.
Для получения информации о ценах необходимо с методом POST обратиться по URL: https://cdis.russvet.ru/rs/massprice

5.	API Остатки
5.1	Попозиционная выгрузка остатков

Для получения попозиционной информации об остатках необходимо с методом GET обратиться по URL:
https://cdis.russvet.ru/rs/residue/*id склада*/*код товара Русского Света*
Пример запроса:
https://cdis.russvet.ru/rs/residue/96/286649

5.2	Массовая выгрузка остатков
Внимание! Данный метод предназначен для загрузки остатков на постоянной основе.
Также существует возможность массово получить информацию об остатках. Для этого нужно обратиться с методом GET по URL:
https://cdis.russvet.ru/rs/residue/all/*id склада*

Параметры:
Параметр	Описание
page	Номер страницы. Общее количество страниц можно получить в заголовке ответа
rows	Количество записей, выводимых на странице. Необязательный параметр. Минимальное
значение – 1, максимальное – 200. Если указать значение > 200, или оставить пустым, то по умолчанию будет проставлено значение 200.
category	Категория позиций, по которым необходимо выводить информацию об остатках. Допустимые значения:
•	skl – для вывода информации только по позициям, которые относятся к складским (т.е. постоянно есть в наличии)
•	all – для вывода информации об остатках по всем позициям, по которым на выбранном складе ненулевые остатки.
Необязательны параметр, если его не заполнять, по умолчанию проставится значение skl.

Пример запроса: https://cdis.russvet.ru/rs/residue/all/96?page=1&rows=100&category=all

CODE	Артикул позиции Русского Света	/residues/	string

RESIDUE	Свободное количество товара на складе РС	/residues/	number

UOM_OKEI	Код единицы измерения по классификатору ОКЕИ	/residues/	string
