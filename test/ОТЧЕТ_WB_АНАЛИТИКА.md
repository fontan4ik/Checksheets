# 🔧 ОТЧЕТ: Исправление WB Аналитики

## 📋 Проблема

**Артикул 23350-1 (nmId: 320893058)**
- ❌ Показывает: 124 заказа
- ✅ Фактически: 132 заказа
- 🔴 Разница: **8 заказов потеряно!**

## 🔍 Корневая причина

**Текущая логика (НЕПРАВИЛЬНАЯ):**
```javascript
// Агрегация по supplierArticle с усечением суффикса
ordersMonth.forEach(order => {
  const art = order.supplierArticle;  // "23350-1"
  const baseArt = String(art).split('-')[0];  // "23350"
  monthStats[baseArt]++;
});

// Чтение из колонки A (артикул продавца)
const articles = sheet.getRange(2, 1, lastRow - 1).getValues().flat();
```

**Проблема:**
- Если есть несколько строк: `23350-1`, `23350-2`, `23350-3`
- Все они получают **одно и то же** значение для базового артикула `23350`
- Заказы распределяются **неравномерно** между nmId

## ✅ Исправленная логика

```javascript
// Агрегация по nmId (артикул WB)
ordersMonth.forEach(order => {
  const nmId = order.nmId;  // 320893058
  monthStats[nmId]++;
});

// Чтение из колонки T (Артикул WB / nmId)
const nmIds = sheet.getRange(2, 20, lastRow - 1).getValues().flat();
```

## ⚠️ Важное примечание

**Почему разница 8 заказов?**

Скорее всего:
1. У вас несколько строк с артикулами `23350-*`
2. WB API возвращает разные количества для каждого nmId
3. Старая логика "сглаживала" эти различия
4. Новая логика показывает точные данные для каждого nmId

**После исправления:** Каждая строка получит **точное** количество заказов для своего nmId.
