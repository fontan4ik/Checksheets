# Инструкция по локальной выгрузке остатков

Эти скрипты позволяют выгружать остатки из ETM и РС локально и обновлять их в Google Таблице.

## 1. Установка зависимостей
Для работы скриптов требуется Python 3 и следующие библиотеки:
```bash
pip install requests gspread google-auth
```

## 2. Настройка `config.py`
Откройте файл `config.py` и установите ID вашей Google Таблицы:
```python
SPREADSHEET_ID = "ВАШ_ID_ТАБЛИЦЫ"
```
ID таблицы можно взять из её URL: `https://docs.google.com/spreadsheets/d/ВАШ_ID_ТАБЛИЦЫ/edit`

## 3. Запуск скриптов
Запустите скрипты из командной строки:

### Обновление ETM (лист "тест"):
```bash
python etm_sync_local.py
```
Скрипт берет артикулы из колонки **A**, запрашивает остатки "Стройкерамика" и записывает их в колонку **AL** ("ЭТМ Стройкерамика").

### Обновление RS (лист "РуСВ TR"):
```bash
python rs_sync_local.py
```
Скрипт: 
1. Загружает весь каталог РС для сопоставления артикулов производителя (Модель) с кодами РС.
2. Обновляет колонки "Остаток АПИ", "Охлад" и "Округление".

### Обновление Feron (лист "ТЕСТ"):
```bash
python feron_sync_local.py
```
Скрипт обновляет колонки:
- **AI**: Ферон Самара
- **AJ**: Ферон Внуково
- **AK**: Ферон Новосибирск

### Запуск всех выгрузок сразу:
```bash
python sync_master.py
```

## Файлы
- `etm_sync_local.py`: Логика работы с ETM.
- `feron_sync_local.py`: Логика работы с Feron.
- `rs_sync_local.py`: Логика работы с РС (использует прокси для обхода блокировок).
- `sync_master.py`: Скрипт для запуска всех выгрузок по очереди.
- `gsheets_utils.py`: Общие функции для работы с Google Таблицами.
- `config.py`: Настройки и учетные данные.
- `seo-pipline-faac250d5e57.json`: Ключ сервисного аккаунта.
